# The Couch Economy

**Wealth, Inactivity, and the Global NCD Burden**

An interactive Shiny app exploring why the world's wealthiest countries are often its most physically inactive — and what that means for cardiovascular disease, diabetes, and obesity rates globally.

---

## Pipeline Documentation

### 1. Where the data comes from

| Source | Indicator | Endpoint |
|--------|-----------|----------|
| WHO Global Health Observatory | Physical inactivity prevalence, adults 18+ (age-standardised), by country, sex, year | REST API — `https://ghoapi.azureedge.net/api/NCD_PAC` |
| World Bank | GDP per capita (USD), Gini coefficient | `wbstats` R package — indicators `NY.GDP.PCAP.CD`, `SI.POV.GINI` |
| Our World in Data | Cardiovascular mortality, diabetes prevalence, obesity prevalence | Programmatic CSV — append `.csv` to any OWID chart URL |
| OpenAI API | Country brief generation | `https://api.openai.com/v1/chat/completions` (gpt-4o-mini) |

### 2. How the data is ingested

`R/fetch_data.R` handles all ingestion:

- **WHO GHO** — queried via the `ghost` R package (`gho_data("NCD_PAC")`). Returns JSON parsed to a data frame. No authentication required.
- **World Bank** — queried via the `wbstats` package (`wb_data()`). Returns a tidy data frame by country-year. No authentication required.
- **OWID** — three CSV URLs fetched with `readr::read_csv()`. No authentication required.
- **OpenAI** — called on demand from the Shiny app via `httr2` POST. Requires `OPENAI_API_KEY` in `.env`.

Raw outputs are saved to `data/raw/` as `.rds` files.

### 3. How the data is processed

`R/process_data.R` handles cleaning and joining:

- WHO sex codes recoded to readable labels (BTSX → Both, MLE → Male, FMLE → Female)
- World Bank Gini gaps filled forward/backward within country using `tidyr::fill()`
- GDP per capita log-transformed for scatter plot scaling
- ISO 3166 country codes standardised across all three sources using the `countrycode` package
- All three sources joined by `iso3 + year` into a single long-format data frame
- Country display names, regions, and continents added via `countrycode`
- Anomaly detection: inactivity values more than 3 SD from the mean are flagged in the log
- Output saved to `data/processed/couch_economy.rds`

Country briefs are generated by `R/generate_brief.R`:

- Data for the selected country is summarised into a structured prompt
- Prompt sent to OpenAI gpt-4o-mini via `httr2`
- Response cached to `data/briefs/` keyed by country + data hash + tone
- Cache invalidated after each weekly data refresh

### 4. What the output is

A Shiny app (`app.R`) with two tabs:

**Explorer tab:**
- Value boxes: number of countries, average inactivity rate, most inactive country
- Scatter plot: GDP per capita (log) vs physical inactivity (%), sized by NCD outcome, coloured by income group — built with `ggplot2` + `plotly`
- Trend chart: inactivity and selected NCD outcome over time for a chosen country
- Country brief: AI-generated 200-word plain-language analysis (policy or general-public tone)

**About tab:** data sources, pipeline description, course context.

### 5. How to run it

**Requirements:** R 4.2+, RStudio (recommended)

**Step 1 — Install packages**

```r
install.packages(c(
  "shiny", "bslib", "dplyr", "ggplot2", "plotly", "scales", "here", "glue",
  "httr2", "jsonlite", "wbstats", "readr", "tidyr", "purrr",
  "countrycode", "digest"
))
```

**Step 2 — Set up environment**

```bash
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY
```

**Step 3 — Fetch and process data**

```r
source("R/fetch_data.R")    # ~2 min, requires internet
source("R/process_data.R")  # ~5 sec
```

**Step 4 — Launch the app**

```r
shiny::runApp()
```

**Deployment to shinyapps.io:**

```r
install.packages("rsconnect")
rsconnect::setAccountInfo(name="...", token="...", secret="...")
rsconnect::deployApp(appName = "couch-economy")
```

Add `SHINYAPPS_NAME`, `SHINYAPPS_TOKEN`, `SHINYAPPS_SECRET`, and `OPENAI_API_KEY` as GitHub repository secrets for automated deployment.

## Data Notes

- WHO inactivity data is available for most countries but only for select years (typically 2000, 2010, 2016, 2022). The pipeline joins on exact year matches.
- World Bank Gini data is sparse; gaps are filled forward/backward within each country.
- OWID data coverage varies by indicator. Missing values are displayed as gaps in trend charts rather than imputed.
- No data contains protected health information. Synthetic data is not required.
